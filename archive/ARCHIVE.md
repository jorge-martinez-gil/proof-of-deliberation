# Archived Scripts

This directory preserves earlier iterations of the experiment runners that
predate the consolidated `src/pod/` package. They are retained for full
historical transparency (as required by the IEEE TKDE reproducibility policy)
but **must not be used to reproduce the results reported in the paper**.

The canonical, paper-aligned entry points live under
[`src/pod/`](../src/pod/) and are exposed through the `pod-experiments` and
`pod-validate` console scripts (see [README.md](../README.md) and
[docs/REPRODUCIBILITY.md](../docs/REPRODUCIBILITY.md)).

## Contents

| File | Origin | Why archived |
|------|--------|---------------|
| `pod-unified2.py` | Intermediate refactor of `pod-unified.py` with additional configuration overrides. | Superseded by the modular `pod.experiment` API. |
| `pod_elec.py` | Earlier Electricity-only variant predating the unified runner. | Functionality is fully covered by `pod-experiments --datasets elec2`. |
| `pod_unified (1).py` | A copy generated by the OS during download; contains the same code path as `pod-unified.py` with minor parameter drift. | Duplicate, kept for diff auditing only. |
| `hick_hyman_validation.py` | Version 1 of the Hick-Hyman validation script (online learning, uncalibrated). | Superseded by `hick_hyman_pod_validation_v2.py` (OOF + isotonic + temperature scaling), which is now packaged as `pod.validation`. |
| `pod-unified.py` | Pre-refactor flat-layout entry point. | Byte-exact numerical equivalence with the new `pod-experiments` console script is asserted by the verification step in `docs/REPRODUCIBILITY.md`; the modular package is the supported way to reproduce the paper. |
| `hick_hyman_pod_validation_v2.py` | Pre-refactor flat-layout entry point for the Hick-Hyman validation. | Functionality fully covered by `pod-validate` and `pod.validation`. |

## Reproducing legacy figures

The archived scripts remain runnable on a system that satisfies the
dependencies in `../requirements.txt`, but no guarantees are made: command-line
flags, default seeds, and output paths differ from the canonical runner.

If you need to reproduce a specific figure that only the archived runner
produced, please open an issue.
